# AI 投研工作规范

本目录用于沉淀具体标的、行业或主题的投研记录。AI 在这里工作时，应优先把研究过程落到 Markdown 文件中，而不是只在对话里给结论。

## 基本原则

- 投研必须围绕“能否形成投资决策”展开，避免只做资料堆砌。
- 所有结论都要区分事实、推断和个人判断。
- 对最新行情、公告、财报、政策、估值、公司动态等时效性信息，必须联网核验，并尽量引用原始来源。
- 不确定的信息要明确标注，不要用模糊记忆替代验证。
- 投资建议需要包含风险和反面观点，不能只写看多逻辑。

## 新建投研文件流程

当用户要求研究某个公司、股票、行业或主题时，AI 应按以下流程执行：

1. 读取 `模版.md`。
2. 手动复制 `模版.md` 的完整内容。
3. 新建对应的投研文件，并将模板内容粘贴进去。
4. 将模板中的 `{{标的名称}}`、`{{YYYY-MM-DD}}`、`{{公司/股票/行业/主题}}`、`{{一句话说明所属行业、业务或主题}}` 等占位符替换为具体内容。
5. 根据文件标题同步更新“目录”中的锚点链接，保证 Obsidian 可以点击跳转。
6. 在新文件中基于模板逐项开展投研和填写内容。
7. 如果用户没有指定文件名，按以下规则命名：
   - 公司/股票：`公司名.md`，例如 `贵州茅台.md`。
   - 行业：`行业名行业.md`，例如 `光伏行业.md`。
   - 主题：`主题名.md`，例如 `AI 算力.md`。
8. 如果同名文件已存在，不要覆盖。应先读取现有内容，并在原文件中补充或更新；如果需要另起版本，使用日期后缀，例如 `公司名-2026-06-09.md`。

## Markdown 排版规范

- 投研文件必须使用 Obsidian 友好的层级和排版。
- 文件第一行使用一级标题：`# 标的名称 投研记录`。
- 文档开头必须包含元信息块：
  - `更新日期`
  - `标的`
  - `定位`
- 元信息块必须包含 `市场` 与 `行业` 标签，用于网页分组展示，例如 `> 市场：美国`、`> 行业：半导体`。也可用 `> 标签：美国,半导体` 一行写法。
- 元信息块后使用 `---` 分隔，再放“目录”。
- “目录”本身使用一级标题 `# 目录`，并从最大的 `# 标题` 开始列出文档结构。
- 主要大段落使用一级标题 `#`，固定为：
  - `# 核心判断`
  - `# 资料来源`
  - `# 基本面`
  - `# 财务与估值`
  - `# 投资框架`
- 主要大段落下的小节使用二级标题 `##`，不要再使用 `####` 作为主小节标题。
- 大段落之间用单独一行 `---` 分隔。
- 一、二级标题后保留一个空行再写正文、表格或列表；三级小标题（###）与其下方正文或列表之间不要留空行。
- 表格前后必须各保留一个空行。
- 表格不能缩进到列表里；表格行必须从行首开始写 `|`，否则 Obsidian 可能无法渲染。
- 适合结构化比较或多维对照的内容优先用表格，例如：
  - 相关链接（项目 / 链接/内容 / 说明）
  - 公司竞争力（维度 / 事实判断 / 跟踪点）
  - 经营分析中的营收构成（分部/项目 / 营收 / 同比 / 占比说明）单独成表
  - 经营分析中的成本构成（项目 / 金额 / 占营收比 / 说明）单独成表
  - 经营分析的其他经营要点（项目 / 当前情况 / 判断）
  - 财务分析及其补充判断
  - 当前估值（TTM P/S 市销率独立成行）
  - 同业估值对比（填入竞争对手的确切财务数据与估值倍数）

- 以叙述、清单或决策逻辑为主的内容用列表或小标题，不要硬塞进表格，例如：
  - 投资决策（使用三级标题 `###` 分列动作、仓位、持有周期、核心理由、最大风险）
  - 公司概况（使用三级标题 `###` 分列所在行业、主营业务、商业模式、客户/渠道、集中度、管理层、持股等）
  - 行业分析（使用三级标题 `###` 分列行业空间、增长驱动、竞争格局、周期性、政策/监管影响）
  - 估值判断
  - 关键假设
  - 相关风险
  - 自己想法
  - 未来关注
  - 决策
- 长列表不要使用两级无序列表，如果需要更清晰的层级，一级使用有序列表，二级使用无序列表，且一级条目之间要留空行，方便 Obsidian 阅读模式扫读。例如：

```markdown
1. 行业地位：
   - 具体事实 1。
   - 具体事实 2。

2. 护城河：
   - 具体事实 1。
   - 具体事实 2。
```

- 不要把一整段密集嵌套列表连续堆在一起；一级条目之间要留空行，方便 Obsidian 阅读模式扫读。
- 内容要短句化。表格单元格可以写完整判断，但不要塞进过长的资料堆砌。
- 若使用推断，必须明确写“推断”；若是个人观点，放在“自己想法”或“估值判断”中。

## 投研填写规范

- 内容应尽量简短概括，优先写关键结论、关键数据和关键分歧，避免长篇铺陈。
- 适合对比、拆分、跟踪的内容应尽量使用表格展示，方便人类快速阅读和复盘。
- `一句话结论`：用一句话说明当前判断，例如“观察，估值不便宜但基本面仍在改善”。
- `投资决策`：使用三级标题 `###` 分列动作、仓位、持有周期、核心理由和最大风险，每项标题后写一句话结论，不使用表格。
- `相关链接`：使用三级表格形式展示（项目 / 链接/内容 / 说明），优先放公司公告、财报、交易所披露、监管文件、公司官网、权威数据源。
- `公司概况`：使用三级标题 `###` 分列所在行业、主营业务、商业模式、主要客户/渠道、客户集中度、管理层、持股比例，每项标题后写短句说明，不使用表格。重点说明公司到底靠什么赚钱，客户是谁，商业模式是否稳定。
- `行业分析`：使用三级标题 `###` 分列行业空间、增长驱动、竞争格局、周期性、政策/监管影响，每项标题后写短句说明，不使用表格。
- `公司竞争力`：不要只写“龙头”，要说明龙头地位来自成本、技术、渠道、品牌、规模还是牌照。
- `经营分析`：关注营收结构、成本结构、量价变化、客户集中度、产能和库存。营收构成和成本构成必须各自单独成表，不要并到同一行或同一张表里；营收构成按分部/产品/地区拆分并标注同比和口径（如分部含对内收入、不可简单相加要说明），成本构成按 COGS、毛利、研发、销售/管理、一次性/重组项、经营利润拆分并给占营收比，若数据为由毛利率等反推必须标注“反推/推断”。
- `财务分析`：至少关注利润率、现金流、ROE/ROIC、债务、资本开支、分红/回购。
- `估值分析`：必须包含当前估值（需将市销率 TTM P/S 独立成行）、历史区间、同业对比（填入竞争对手的确切营收、市值与 P/S 等估值数据，避免使用模糊定性词汇）、合理区间、上行和下行空间。
- `关键假设`：写清楚投资判断依赖哪些可验证假设。
- `相关风险`：覆盖业务、财务、政策、竞争、估值和极端风险。
- `未来关注`：写成可跟踪指标或事件，不要只写“继续关注业绩”。
- `决策`：最后落到明确动作，并留下复盘记录入口。

## 信息来源规范

- 优先使用一手资料：公告、年报、季报、招股书、交易所问询函、监管文件、公司官网。
- 二手资料可以辅助理解，但不能替代关键事实来源。
- 使用研报、媒体、论坛、社交平台信息时，需要标明其性质，并避免把观点当事实。
- 涉及当前股价、市值、估值、财报日期、股东变化、政策变化等信息时，需要核验日期。
- 引用数据时尽量写明时间范围，例如“2025 年年报”“截至 2026-06-09”。

## 输出习惯

- 先把研究内容写入对应 Markdown 文件，再向用户总结。
- 文件内容优先使用短句、要点和表格，让读者可以快速扫读。
- 写完后必须快速检查：
  - 是否存在以空格开头的表格行。
  - 是否仍有 `####` 作为主要章节标题。
  - 目录锚点是否与标题一致。
  - 嵌套列表的一级条目之间是否有空行。
- 总结时说明：
  - 创建或更新了哪个文件。
  - 当前初步结论是什么。
  - 哪些信息已经核验。
  - 哪些问题仍需继续跟踪。
- 不要在没有写入文件的情况下，只给一段临时投研结论，除非用户明确要求只在对话中回答。

## 风险提示

本目录内容用于投研记录和决策辅助，不构成确定性收益承诺。任何投资结论都应保留复盘和修正空间。
